# MindKernel v0.2 计划：Daemon 实时对话观测与记忆提炼

_日期：2026-02-26（Asia/Shanghai）_

## 0) 结论先行

- v0.1 已闭环，不依赖 daemon 也可稳定运行。
- v0.2 建议引入 **可开关 daemon（默认关闭）**：用于把“对话事件→记忆提炼→治理入队”从批处理升级到准实时。
- 原则：**不替代现有 scheduler/worker，只做事件观测与投递层**，确保可回退。

---

## 1) 目标与非目标

### 目标
1. 对话事件进入后，分钟级完成记忆提炼候选（retain/reflect）。
2. 保持现有治理链路（Gate / DecisionTrace / AuditEvent）不破坏。
3. 提供可观测指标：freshness、drop rate、duplicate rate、queue lag。

### 非目标
1. 不在 v0.2 引入“强制实时写回”。
2. 不替代 `tools/scheduler/*` 的执行职责。
3. 不强依赖向量检索（当前结论仍 `NO_GO_KEEP_FTS`）。

---

## 2) 目标架构（增量）

```text
Conversation Events
   -> daemon observer (tail/stream)
      -> normalize + dedupe + throttle
         -> candidate extractor (retain/reflect proposals)
            -> enqueue scheduler jobs (reflect/verify/revalidate)
               -> existing workers + gate + audit
```

关键点：
- daemon 只负责“检测 + 投递”，执行仍由现有 worker 完成。
- 入队幂等键规范化：`session_id + turn_id + candidate_hash`。
- 支持 backpressure：队列高水位时自动降频，仅保留高价值候选。

## 3) 工作分解（D1~D6）

### D1. Daemon Skeleton（基础骨架）
- 产物：`tools/daemon/memory_observer_daemon_v0_2.py`
- 能力：
  - 支持 `--mode tail|poll`（先从稳定模式开始）
  - PID/lock 文件、优雅退出、重启续跑（checkpoint）
  - state db（offset/checkpoint）
- 验收：重启不重复消费，不丢事件。

### D2. 事件标准化与去重节流
- 产物：`core/event_normalizer_v0_2.py`
- 能力：
  - 统一事件结构（session/channel/role/content/tool_call）
  - dedupe window（例如 5 分钟）
  - throttle（每会话每分钟上限）
- 验收：重复率下降且无关键事件漏报。

### D3. 候选提炼与治理入队
- 产物：`core/realtime_memory_candidate_v0_2.py`
- 能力：
  - 从对话实时提取 retain/reflect 候选
  - 生成 idempotency key 并入 `scheduler_jobs`
  - 风险分层（低/中/高）与 gate 对齐
- 验收：入队后 worker 可无改造消费。

### D4. 观测与告警
- 产物：
  - `tools/validation/validate_daemon_memory_observer_v0_2.py`
  - 指标落盘：`reports/daemon/*.json`
- 指标：
  - freshness p95
  - duplicate drop rate
  - enqueue success rate
  - queue lag p95
- 验收：指标可持续采集并纳入 weekly report。

### D5. 运行策略与回退
- 产物：`docs/06-execution/daemon-runbook-v0.2.md`
- 能力：
  - 默认关闭（feature flag）
  - 分阶段灰度：dev -> shadow -> partial sessions -> default-on
  - 一键回退到“仅批处理”
- 验收：任意阶段可在 1 分钟内回退。

### D6. CI 与发布门禁
- 产物：
  - 新增 daemon 相关验证脚本到 release_check（quick/full 分层）
  - 可选 nightly daemon 回放任务
- 验收：主门禁时长可控，不牺牲覆盖。

---

## 4) DoD（完成定义）

满足以下即判定 v0.2 daemon MVP 完成：
1. Daemon 连续运行 24h 无崩溃；重启可恢复。
2. 对话到候选入队 freshness p95 < 120s。
3. 重复消费率 < 1%，漏提炼可追踪。
4. 失败全部有审计痕迹（error_code + trace_id）。
5. 可在不影响现有 worker 的前提下一键关闭 daemon。

---

## 5) 风险与缓解

1. **噪音放大风险**：实时提炼可能增加低价值候选
   - 缓解：value score + throttle + 高水位降频
2. **并发重复风险**：多实例 daemon 重复入队
   - 缓解：instance lock + idempotency key + enqueue 去重
3. **观测盲区风险**：实时链路失败不易发现
   - 缓解：强制审计 + 健康探针 + 每日 summary
4. **门禁时长膨胀**
   - 缓解：daemon 长时回放放 nightly，主门禁保留核心快检
